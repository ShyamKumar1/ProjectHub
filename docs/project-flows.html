    🔐 Authentication

    OAuth · JWT · HTTP-only cookies

    OAuth-only auth via Google (primary) and GitHub (secondary). JWT stored in HTTP-only cookie token with 7-day expiry. Bearer token fallback for API calls. Rate-limited at 100 req/min.

    🔑 Google Login

    Trigger: User clicks "Continue with Google" on /login
    1. LoginPage generates PKCE challenge (SHA-256 code_verifier)
    2. Stores PKCE verifier in sessionStorage('pkce_verifier')
    3. Redirects to Google OAuth URL (code_challenge_method=S256)
    4. Google redirects back to /api/auth/google/callback?code=xxx
    5. GoogleCallback reads code + pkce_verifier from sessionStorage
    6. POSTs { code, code_verifier, redirect_uri } → /api/v1/auth/google
    7. Fastify exchanges code for access_token with Google's token endpoint
    8. Fetches user info from Google People API → creates/links user in DB
    9. Signs JWT, sets HTTP-only cookie (httpOnly, sameSite=lax, path=/, 7d)
    10. Returns { token, user } → callback stores Bearer token → redirects to /dashboard
    login/page.tsx api/auth/google/callback/page.tsx routes/auth.ts (POST /auth/google) store/auth.ts

    🐙 GitHub Login

    Trigger: User clicks "Continue with GitHub" on /login
    1. Redirects to GitHub OAuth URL
    2. GitHub redirects to /api/auth/github/callback?code=xxx
    3. GithubCallback calls loginWithGithub(code) from auth store
    4. POSTs { code } → /api/v1/auth/github
    5. Fastify exchanges code with GitHub API → creates/links user
    6. Sets HTTP-only cookie + returns { token, user }
    7. Redirects to /dashboard
    api/auth/github/callback/page.tsx routes/auth.ts (POST /auth/github)

    🔍 Session Check

    Trigger: Every protected page load (AppContent useEffect)
    1. AppContent calls checkSession() (skipped on /login and /api/auth/*)
    2. checkSession → GET /api/v1/auth/session
    3. Fastify reads HTTP-only cookie, verifies JWT, fetches user from DB
    4. If valid: returns { user, expires } → user set in Zustand store ✅
    5. If 401: user stays null → routing effect redirects to /login
    layout.tsx (AppContent) store/auth.ts (checkSession) routes/auth.ts (GET /auth/session)

    🚪 Logout

    Trigger: User clicks logout button in Navbar
    1. Navbar calls logout() from auth store
    2. Sets ph_logged_out flag in sessionStorage
    3. Clears Bearer token via api.setToken(null)
    4. Sets user = null in Zustand store
    5. POST /api/v1/auth/logout → Fastify clears HTTP-only cookie (Max-Age=0)
    6. window.location.href = '/login' (full page navigation)
    7. Login page: clears flag → calls checkSession → 401 → stays on login
    Navbar.tsx store/auth.ts (logout) routes/auth.ts (POST /auth/logout)

    📁 Project Management

    CRUD · Members · Status

    ➕ Create Project

    Trigger: User clicks "New Project" in Navbar or dashboard
    1. Navbar sets createProjectOpen = true in UI store
    2. Dashboard renders Modal with form (name, description, icon, status, visibility)
    3. User fills form → clicks "Create Project"
    4. POST /api/v1/projects → Fastify creates project + logs project_created activity
    5. Returns 201 → dashboard refetches project list, closes modal
    dashboard/page.tsx routes/projects.ts (POST /projects) db/models.ts (createProject)

    📄 View Project Detail

    Trigger: User clicks a project card → /projects/:id
    1. ProjectDetailPage mounts → GET /api/v1/projects/:id
    2. Fastify fetches project + members + tasks + resources + activity + linked_repo
    3. Returns all data in a single response
    4. Page renders: hero header, contribution graph, tabbed content
    projects/[id]/page.tsx routes/projects.ts (GET /projects/:id)

    ✏️ Update / 🗑️ Delete Project

    Trigger: Edit button or trash icon on project card
    1. Edit: opens modal → PATCH /api/v1/projects/:id → updates DB (owner/admin/editor)
    2. Delete: confirm → DELETE /api/v1/projects/:id → owner only → cascading delete
    projects/[id]/page.tsx ProjectCard.tsx routes/projects.ts (PATCH|DELETE /projects/:id)

    ✅ Task Management

    List & Kanban · Subtasks · Priority

    Tasks belong to projects, ordered by position (auto-incrementing). Support subtasks (self-referencing parent_task_id), priority P0-P3, due dates, assignment, and status lifecycle. Two views: List (tree with expand/collapse) and Kanban (drag-between-columns).

    ➕ Create Task

    1. Click "Add Task" in Tasks tab → modal with title, description, priority, due_date
    2. POST /api/v1/projects/:id/tasks
    3. Fastify creates task with auto-position (MAX(position) + 100)
    4. Returns 201 → task list refreshes
    TaskList.tsx routes/tasks.ts (POST /projects/:id/tasks)

    🔄 Toggle Status

    1. Click circular checkbox → status: 'completed' | 'pending'
    2. PATCH /api/v1/tasks/:taskId
    3. Fastify updates + logs task_completed activity (double-check for StrictMode)
    TaskList.tsx (handleToggleStatus) routes/tasks.ts (PATCH /tasks/:taskId)

    📋 Kanban Move

    1. Kanban card shows quick-action buttons → click a target status
    2. PATCH /api/v1/tasks/:taskId with { status: newStatus }
    3. Card moves to the new column on refresh
    KanbanBoard.tsx

    ⚠️ Known Issues & Fixes

    Recent patches

    🔄 API Proxy via Rewrites

    Change: lib/api.ts now uses API_BASE = '' — all requests go same-origin

    Previously the browser made CORS requests directly to localhost:4000 using NEXT_PUBLIC_API_URL. Now next.config.js has a rewrites() rule proxying /api/v1/:path* to the Fastify backend. This eliminates all CORS complexity — no preflight requests, no credential headers needed.

    next.config.js (rewrites) lib/api.ts (API_BASE = '')

    📐 Server Components for Redirects

    Change: page.tsx and projects/page.tsx → server components

    Changed from 'use client' with useRouter to server components using Next.js redirect(). Eliminates client-side hydration flash and unnecessary JS bundle for redirect-only pages.

    app/page.tsxapp/projects/page.tsx

    🎨 Tailwind Dynamic Classes Fixed

    Change: Dashboard stat colors use colorClass instead of color

    Tailwind CSS cannot tree-shake dynamically constructed classes like text-${stat.color}. Changed to colorClass: 'text-accent' (full literal class names), so Tailwind's JIT compiler can detect and include them.

    dashboard/page.tsx

    📊 Contribution Graph Month Labels

    Change: ContributionGraph.tsx fixed month label rendering

    Old code used .filter((_, i) => i % 4 === 0) with broken destructuring. Now properly maps to { week, weekIndex } and calculates labels via Math.floor(weekIndex / 4.3) % 12 for accurate 52-week alignment.

    ContributionGraph.tsx

    ⚙️ Streak Preferences Controlled

    Change: Settings page streak checkboxes are now controlled components

    Streak preferences in Settings now use useState with checked prop and onChange handlers that immediately persist via api.updateProfile({ streak_preferences }). Previously used defaultChecked which lost state on re-render.

    settings/page.tsx

    🕒 ProjectCard Null Date Guard

    Change: getTimeAgo() handles null dates

    Added if (!dateStr) return ''; guard to prevent new Date(null).getTime() returning NaN, which caused Invalid Date rendering in project cards with missing timestamps.

    ProjectCard.tsx

    🔑 Friends List Unique Key

    Change: Pending requests card key uses composite key

    Changed key from r.user_id_1 (which can collide) to `${r.user_id_1}-${r.user_id_2}` to ensure unique React keys when multiple pending requests exist.

    friends/page.tsx
